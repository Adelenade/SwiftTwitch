Analytics

public struct Analytics

Analytics is a category of Twitch API calls that provide insight to the game usage or extension usage of the token bearing user.

  • AnalyticsType

    type defines the different types of extension analytics reports. overviewVersion1 returns analytics for the first 90 days, and overviewVersion2 returns analytics from January 31st, 2018 to the current day.

    These reports are viewable and downloadable via a URL that is returned as a response.

    A complete comparison is available here

    • overviewVersion1: The first version of extension analytics reports
    • overviewVersion2: The second version of extension analytics reports.

  • GetExtensionAnalyticsResult

    GetExtensionAnalyticsResult defines the different types of results that can be retrieved from the getExtensionAnalytics call of the Analytics API. Variables are included that specify the data that was returned.

    • success: Defines that the call was successful. The output variable will contain all extension analytics data.
    • failure: Defines that the call failed. Returns all data corresponding to the failed call. These data pieces are as follows:
    • Data? - The data that was returned by the API
    • URLResponse? - The response from the URL task
    • Error? - The error that was returned from the API call

  • getExtensionAnalytics(tokenManager:after:startedAt:endedAt:extensionId:first:type:completionHandler:)

    getExtensionAnalytics will run the Get Extension Analytics API call of the New Twitch API. You can use this API call to retrieve data regarding the usage of extensions for the token-bearing user. This data will be available for download in a URL that is retrieved from this API call.

    This API call requires a token with analytics:read:extensions permissions.

    More information about the web call is available here

    Parameters

    tokenManager

    The TokenManager whose token should be used. Singleton by default.
    after

    The pagination token of the call. This parameter is ignored if the extensionId parameter is specified.
    startedAt

    The date after which all analytics should start after. endedAt must also be specified.
    endedAt

    The date before which analytics should be gathered for. startedAt must also be specified.
    extensionId

    The extension to gather analytics for. If this is specified, only the extension with the specified ID will be analyzed.
    first

    The number of objects to retrieve.
    type

    The type of report to gather. For more information, please see documentation on AnalyticsType.
    completionHandler

    The function that should be run whenever the retrieval is successful. There are two types of GetExtensionAnalyticsResult: success and failure.
