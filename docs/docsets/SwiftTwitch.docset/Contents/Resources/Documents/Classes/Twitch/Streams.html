Streams

public struct Streams

Streams is a category of Twitch API calls that interacts with Streams on Twitch.

  • GetStreamsResult

    GetStreamsResult defines the different types of results that can be retrieved from the getStreams call of the Streams API. Variables are included that specify the data that was returned.

    • success: Defines that the call was successful. The output variable will contain all returned data.
    • failure: Defines that the call failed. Returns all data corresponding to the failed call. These data pieces are as follows:
    • Data? - The data that was returned by the API
    • URLResponse? - The response from the URL task
    • Error? - The error that was returned from the API call

  • getStreams(tokenManager:after:before:communityIds:first:gameIds:languages:userIds:userNames:completionHandler:)

    getStreams will run the Get Streams API call of the New Twitch API. The returned streams are sorted in descending order such that the most-watched streamer is returned first in the received data.

    This method does not require a TwitchTokenManager. No Authorization is required.

    More information about the web call is available here

  • getStreamsMetadata(tokenManager:after:before:communityIds:first:gameIds:languages:userIds:userNames:completionHandler:)

    getStreamsMetadata will run the Get Streams Metadata API call of the New Twitch API. The returned streams are sorted in descending order such that the most-watched streamer is returned first in the received data. This data contains special information about streamers who are playing Overwatch or Hearthstone!

    This method does not require a TwitchTokenManager. No Authorization is required.

    More information about the web call is available here

  • createStreamMarker(tokenManager:userId:description:completionHandler:)

    createStreamMarker will run the Create Stream Marker API call of the New Twitch API. Stream Markers are points in streams that the Streamer or Editor wants to mark so that they can return to that point later. The marker will be created at the current timestamp of the livestream.

    This method does requires user:edit:broadcast permissions on a user who is an editor or streamer of the stream where the stream marker should be created.

    More information about the web call is available here
